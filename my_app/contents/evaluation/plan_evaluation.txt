# 🧪 FIREgenerator 추천 시스템 평가 가이드

## 📋 전체 개요

FIREgenerator 금융 지식 추천 시스템의 **LLM Judge + Human Feedback 평가** 프로세스

### 🎯 평가 목표
복잡한 데이터베이스 연결 없이 **CSV 파일만으로 LLM Judge와 Human Feedback**을 통해 추천 시스템의 품질을 평가합니다.

## 📊 2단계 간소화 프로세스

### 1️⃣ LLM Judge 평가 (llm_judge.py)
```bash
python llm_judge.py
```

**목적**: CSV에서 사용자 데이터를 로드하여 LLM으로 추천 품질 평가

**평가 기준** (9가지, 1-4 추천 품질, 5-9 설명 품질):
- **1. 관련성 (Relevance)**: 사용자 관심사와 얼마나 잘 맞는가?
- **2. 적합성 (Suitability)**: 사용자 지식 수준에 적합한가?
- **3. 다양성 (Diversity)**: 추천 콘텐츠들이 다양한 관점을 제공하는가?
- **4 .실용성 (Practicality)**: 실제로 사용자에게 도움이 될 것인가?
- **5. 정확성 (Accuracy)**: 추천이 사실에 부합하는가?
- **6. 완결성 (Completeness)**: 필요한 정보를 모두 포함하고 있는가?
- **7 .일관성 (Coherence)**: 설명이 자연스럽고 논리적인가?
- **8. 안전성 (Safety)**: 유해하거나 편향된 내용을 포함하지 않는가?
- **9. 어조 및 스타일 (Tone & Style)**: 의도한 페르소나를 잘 유지하는가?

**결과 파일**:
- `llm_evaluation_summary_YYYYMMDD_HHMMSS.csv`: LLM 평가 요약
- `llm_evaluation_detailed_YYYYMMDD_HHMMSS.json`: LLM 평가 상세 결과

---

### 2️⃣ Human Feedback 수집 (human_feedback.py)
```bash
python human_feedback.py
```

**목적**: 실제 사람이 LLM Judge 결과를 보고 직접 평가

**프로세스**:
1. 1단계 LLM 결과 로드
2. 각 추천 결과를 콘솔에 보기 좋게 표시
3. 사용자가 1-5점 평가 + 코멘트 입력
4. CSV/JSON으로 결과 저장

**평가 표시 정보**:
- 👤 사용자 정보 (이름)
- 📋 추천된 콘텐츠들 (제목, 내용, 난이도, 태그)
- 🤖 LLM 모델별 점수 + 평가 이유

**결과 파일**:
- `human_feedback_detailed_YYYYMMDD_HHMMSS.json`: 상세 피드백 결과
- `human_feedback_summary_YYYYMMDD_HHMMSS.csv`: Human vs LLM 비교

## 📊 결과 해석 가이드

### LLM vs Human 비교 분석
1. **상관계수 분석**: LLM Judge와 Human 평가가 얼마나 일치하는지 확인
2. **차이 케이스 분석**: 평가가 크게 다른 경우 원인 분석
3. **모델별 성능 비교**: GPT, Gemini, Claude 각각의 평가 특성 파악

### 결과 활용 방법
- **평균 점수**: 전반적인 추천 시스템 품질 수준 파악
- **점수 분포**: 사용자별, 모델별 편차 확인
- **코멘트 분석**: Human Feedback의 정성적 인사이트 수집

### 개선 포인트 도출
```
1. Human 점수가 높지만 LLM 점수가 낮은 경우 → 프롬프트 개선 필요
2. 모든 점수가 낮은 경우 → 추천 알고리즘 전면 검토
3. 점수 편차가 큰 경우 → 일관성 개선 필요
```
