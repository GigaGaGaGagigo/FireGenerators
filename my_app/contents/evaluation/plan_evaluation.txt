# 🧪 FIREgenerator 추천 시스템 평가 가이드

## 📋 전체 개요

FIREgenerator 금융 지식 추천 시스템의 LLM Judge + Human Feedback 평가 프로세스

### 🎯 평가 목표
LLM Judge와 Human Feedback을 통해 추천 시스템의 품질을 평가합니다.

## 📊 2단계 간소화 프로세스

### 1️⃣ LLM Judge 평가 (llm_judge.py)
```bash
python llm_judge.py
```

**목적**: CSV에서 사용자 데이터를 로드하여 LLM으로 추천 품질 평가

**평가 기준** (4가지 핵심 기준):
- **1. 적합성 (Suitability)**: 사용자 지식 수준에 적합한가?
- **2. 관련성 (Relevance)**: 사용자 관심사와 얼마나 관련있는 추천인가?
- **3. 다양성 (Diversity)**: 추천 콘텐츠들이 다양한 관점을 제공하는가?
- **4. 일관성 (Coherence)**: 콘텐츠와 맞춤 설명이 자연스럽고 논리적으로 연결되는가?

**결과 파일**:
- `llm_evaluation_summary_YYYYMMDD_HHMMSS.csv`: LLM 평가 요약
- `llm_evaluation_detailed_YYYYMMDD_HHMMSS.json`: LLM 평가 상세 결과

---

### 2️⃣ Human Feedback 수집 (human_feedback.py)
```bash
python human_feedback.py
```

**목적**: 실제 사람이 LLM Judge 결과를 보고 직접 평가

**프로세스**:
1. 1단계 LLM 결과 로드
2. 각 추천 결과를 Streamlit UI에 보기 좋게 표시
3. 사용자가 LLM과 동일한 4개 기준으로 1-5점 평가 + 코멘트 입력
4. CSV/JSON으로 결과 저장

**Human 평가 기준** (LLM과 동일):
- 적합성, 관련성, 다양성, 일관성 (각 1-5점)

**평가 표시 정보**:
- 👤 사용자 정보 (이름)
- 📋 추천된 콘텐츠들 (제목, 내용, 난이도, 태그)
- 🤖 LLM 모델별 점수 + 평가 이유

**결과 파일**:
- `human_feedback_detailed_YYYYMMDD_HHMMSS.json`: 상세 피드백 결과
- `human_feedback_summary_YYYYMMDD_HHMMSS.csv`: Human vs LLM 비교

## 📊 결과 해석 가이드

### LLM vs Human 비교 분석
1. **상관계수 분석**: LLM Judge와 Human 평가가 얼마나 일치하는지 확인
2. **차이 케이스 분석**: 평가가 크게 다른 경우 원인 분석

### 결과 활용 방법
- **전체 점수**: LLM 제공 또는 4개 기준 평균으로 계산된 종합 점수
- **세부 점수**: 적합성, 관련성, 다양성, 일관성 각각의 점수 분석
- **LLM vs Human 비교**: 동일한 4개 기준으로 평가한 결과 비교
- **코멘트 분석**: Human Feedback의 정성적 인사이트 수집

### 개선 포인트 도출
```
1. Human 점수가 높지만 LLM 점수가 낮은 경우 → 프롬프트 개선 필요
2. 모든 점수가 낮은 경우 → 추천 알고리즘 전면 검토
3. 점수 편차가 큰 경우 → 일관성 개선 필요
4. 특정 기준(적합성/관련성/다양성/일관성) 점수가 낮은 경우:
   - 적합성 낮음 → 사용자 레벨 매칭 로직 개선
   - 관련성 낮음 → 관심사/목표 반영 로직 개선  
   - 다양성 낮음 → 추천 다양성 알고리즘 개선
   - 일관성 낮음 → 설명 생성 프롬프트 개선
```

## 📈 평가 기준 상세 설명

### 🎯 4개 핵심 평가 기준
1. **적합성 (Suitability)**
   - 사용자의 지식 수준(Beginner/Intermediate/Advanced)에 맞는 난이도인가?
   - 전문 용어 사용이 적절한가?
   - 사용자가 이해할 수 있는 수준으로 설명되었는가?

2. **관련성 (Relevance)**
   - 사용자의 관심 카테고리와 얼마나 관련있는가?
   - 사용자의 투자 목표와 부합하는가?
   - 사용자의 감정적 성향을 반영했는가?

3. **다양성 (Diversity)**
   - 추천된 콘텐츠들이 다양한 주제를 다루는가?
   - 서로 다른 관점이나 접근법을 제공하는가?
   - 편향되지 않고 균형잡힌 정보를 제공하는가?

4. **일관성 (Coherence)**
   - 콘텐츠와 맞춤 설명이 논리적으로 연결되는가?
   - 전체적인 추천의 흐름이 자연스러운가?
   - 설명이 명확하고 이해하기 쉬운가?

### 🔄 평가 프로세스 개선점
- **9개 → 4개 기준**: 평가 복잡성 감소, 핵심 품질 요소에 집중
- **LLM-Human 일치성**: 동일한 기준으로 평가하여 정확한 비교 가능
- **overall_score 추가**: LLM이 직접 전체 점수 제공 또는 4개 평균으로 계산
