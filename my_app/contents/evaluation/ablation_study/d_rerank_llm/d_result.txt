## 🧪 실험 설계

### D1~D3: LLM 모델 비교

#### D1: GPT-4o-mini (현재 사용)
- **장점**: 검증된 안정성, 예측 가능한 비용
- **단점**: 속도 보통, OpenAI API 의존성

#### D2: Gemini-Flash  
- **장점**: 빠른 응답속도 (212ms), Google API
- **단점**: 무료 한계 (15 RPM, 1,500 RPD)

#### D3: Claude-3-Haiku
- **장점**: Anthropic 모델, 대안 선택지
- **단점**: 성능 데이터 부족

### 측정 지표
- **응답 속도**: LLM 호출 시간 (ms)
- **관련성 점수**: 컨텍스트 이해도
- **일관성**: 동일 입력 대비 결과 일관성  
- **비용 효율성**: API 호출 비용 대비 성능

## 💡 핵심 결정 사항

### 🎯 최종 선택: **GPT-4o-mini 유지**

#### 선정 근거
1. **무료 한계 고려**: Gemini의 15 RPM 제한은 실사용에 부족
2. **안정성 우선**: 검증된 모델의 일관된 성능
3. **비용 예측성**: 명확한 가격 정책으로 예산 계획 용이
4. **실제 병목**: LLM보다는 **필터링 전략**이 더 중요한 개선점

## 🔧 LLM 리랭킹 메커니즘

### 처리 과정
1. **후보 선별**: 상위 7개 후보만 LLM 평가
2. **컨텍스트 구성**: 사용자 프로필 + 콘텐츠 정보
3. **LLM 평가**: 개인화 적합도 점수 (0.0~1.0)
4. **점수 조합**: `최종점수 = 0.7 × LLM점수 + 0.3 × 수치점수`

### LLM 설정 (GPT-4o-mini)
```python
model="gpt-4o-mini"
temperature=0.3      # 일관성 위해 낮은 값
max_tokens=300       # 리랭킹용 짧은 응답
top_p=0.9           # 적당한 창의성
```
