## 🎯 실험 목적

Ko-SRoBERTa + BGE-M3 병행 사용이 최적인지 체계적으로 검증

### 실험 대상
- **검색 모델**: Ko-SRoBERTa vs BGE-M3 vs Hybrid
- **필터링 전략**: Vector Only vs Vector + Rules
- **리랭킹 가중치**: 5가지 조합
- **LLM 모델**: GPT-4o-mini vs Gemini-Flash vs Claude-3-Haiku  
- **프롬프트 전략**: Generic vs Level-Specific vs Adaptive

## 📊 실험 결과 분석

### 🏆 주요 발견사항:

1. **관련성 최고**: `hybrid + vector_only` 조합
   - 필터링을 완화했을 때 더 많은 추천 생성 (5개)
   - 관련성 점수 1.000 (완벽)

2. **속도 최고**: `hybrid + vector_plus_rules + gemini-flash`
   - 평균 응답시간 212ms (가장 빠름)
   - LLM 모델 차이가 속도에 영향

3. **다양성 최고**: `ko-sroberta` 단독 사용
   - 개별 모델이 다양성 측면에서 우수

### 🔍 핵심 인사이트:

1. **필터링 전략의 중요성**:
   - `vector_plus_rules`: 엄격한 필터링으로 추천 수 감소 (평균 0.2-0.4개)
   - `vector_only`: 관대한 필터링으로 더 많은 추천 (5개)

2. **모델별 성능 차이**:
   - `hybrid` > `ko-sroberta` > `bge-m3` 순으로 성능
   - 하이브리드 방식이 속도와 관련성에서 우수

3. **LLM 모델 영향**:
   - `gemini-flash`가 응답 속도 최고
   - LLM 차이가 성능에 큰 영향을 미치지 않음

## 💡 권장사항

현재 **Ko-SRoBERTa + BGE-M3 병행 사용**이 적절하지만, 다음 최적화 고려:

1. **필터링 완화**: `vector_only` 전략 적용으로 추천 수 증가
2. **하이브리드 유지**: 속도와 관련성 균형점에서 최적
3. **Gemini-Flash 활용**: LLM 처리 속도 개선
