# 🧪 FIREgenerator 추천 시스템 평가 프레임워크

## 📊 전체 실험 구조

### 체계적 Ablation Study: A-E 단계별 최적화

```
📂 evaluation/
├── a_embed/         # A. 검색 모델 비교
├── b_hybrid/        # B. 하이브리드 필터링  
├── c_rerank_rule/   # C. 룰 기반 리랭킹 계수
├── d_rerank_llm/    # D. LLM 컨텍스트 리랭킹
├── e_explanation/   # E. 프롬프트 전략
└── final_result/    # 최종 통합 결과
```

## 🎯 단계별 실험 목적

### A. 검색 모델 비교 (완료 ✅)
**목적**: Ko-SRoBERTa vs BGE-M3 vs Hybrid 임베딩 성능 비교  
**결과**: **Hybrid 방식이 최적** (관련성 1.0, 추천 수 5개)

### B. 하이브리드 필터링 (진행중 🚧)  
**목적**: 벡터만 vs 벡터+룰 후보군 수집 비교  
**결과**: **하이브리드 방식이 안정적** (100% 성공률, 다양성 확보) 다만, FAISS 임포트 문제가 있어 해결 후 다시 진행 필요 

### C. 룰 기반 리랭킹 계수 (계획 중 🚧)
**목적**: α(벡터), β(레벨), γ(태그) 가중치 최적화  
**현재**: α=0.6, β=0.3, γ=0.1

### D. LLM 컨텍스트 리랭킹 (결정 완료 ✅)
**목적**: GPT-4o-mini vs Gemini-Flash vs Claude-3-Haiku 비교  
**결정**: **GPT-4o-mini 유지** (안정성 + 비용 효율성)

### E. 프롬프트 전략 (설계 완료 📋)
**목적**: Generic vs Level-Specific vs Adaptive 설명 전략  
**권장**: 레벨별 + 감정별 하이브리드 접근

## 📈 주요 성과 및 결정 사항

### ✅ 완료된 최적화
1. **임베딩 모델**: Ko-SRoBERTa + BGE-M3 하이브리드 유지
2. **필터링 전략**: `level_strict=False`로 완화 (추천 수 증대)  
3. **후보군 수집**: 감정룰 + 벡터 + 기본룰 하이브리드 유지
4. **LLM 모델**: GPT-4o-mini 지속 사용

### 🎯 핵심 개선 결과
- **추천 수**: 0.2개 → 5개 (25배 증가)
- **관련성 점수**: 완벽한 1.0 달성  
- **시스템 안정성**: 100% 성공률 확보
- **다양성**: 최대 다양성 점수 달성

## 📊 실험 방법론

### 데이터셋
- **사용자**: profiles_test DB (21명) + 샘플 (20명)
- **콘텐츠**: Supabase DB (225개)
- **모델**: Ko-SRoBERTa, BGE-M3 임베딩

### 평가 지표
```python
metrics = {
    "추천 수": "생성된 추천 개수",
    "관련성": "사용자 관심사 매칭도",  
    "다양성": "추천 콘텐츠 레벨/카테고리 분산",
    "응답 속도": "전체 처리 시간 (ms)",
    "성공률": "정상 작동 비율 (%)"
}
```

### 실험 환경
- **플랫폼**: Python 3.10, Conda 환경
- **의존성**: FAISS, Supabase, Sentence Transformers
- **하드웨어**: 로컬 개발 환경

```

    ↓

    ↓

    ↓
기본 룰 후보 수집 (10개) 
    ↓
후보군 통합 (최대 30개)
    ↓
수치 기반 리랭킹 (α=0.6, β=0.3, γ=0.1)
    ↓
GPT-4o-mini 컨텍스트 리랭킹 (상위 7개)
    ↓
최종 추천 결과 (3개)
```

**결과**: 안정적이고 개인화된 고품질 추천 서비스 제공 🚀
