## 🧪 실험 설계

### B1: 벡터만 후보군 수집
```python
# 후보군 구성 (FAISS 벡터 인덱스 의존)
candidates = vector_search_results_only  # Ko-SRoBERTa + BGE-M3 벡터 검색만
# 장점: 빠른 처리, 높은 유사도
# 단점: FAISS 의존성, 단일 실패점
```

### B2: 하이브리드 후보군 수집 (현재 시스템)
```python
# 후보군 구성 (다중 검색 전략)
candidates = emotion_rule + vector_search + basic_rule  # 통합
# 장점: 안정성, 다양성, FAISS 실패해도 룰 기반으로 보완
# 단점: 복잡한 처리 파이프라인
```

### 측정 지표
- **추천 수**: 생성된 추천 개수
- **응답 시간**: 처리 속도 (ms)
- **관련성 점수**: 사용자 관심사 매칭도
- **다양성 점수**: 추천 콘텐츠의 레벨 다양성

## 💡 핵심 인사이트

### B2 하이브리드의 장점 (실험으로 검증)
1. **안정성**: FAISS 실패해도 룰 기반으로 추천 계속 가능
2. **다양성**: 감정 + 벡터 + 태그 매칭 다차원 검색  
3. **개인화**: 사용자 감정 상태(-50~+50)와 관심사 반영
4. **확장성**: 새로운 검색 전략 추가 용이
5. **실제 검증**: 100% 성공률로 안정성 확인

### B1 벡터만의 한계 (실험에서 확인)
1. **단일 실패점**: FAISS 인덱스 없으면 완전 실패 (0% 성공률)
2. **환경 의존성**: 임베딩 모델 + 벡터 인덱스 + Python 환경 모두 필요
3. **제한된 범위**: 벡터 공간 내에서만 검색, 룰 기반 보완 없음

## 🎯 결론
**B2 하이브리드 후보군 수집 방식**이 안정성과 다양성 측면에서 우수하며, 현재 시스템의 하이브리드 접근 방식이 올바른 선택임을 확인했습니다.
