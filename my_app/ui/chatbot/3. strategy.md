  def get_base_personality_prompt(user_name: str = "", emotion: str = "중립") -> str:

  사용자 정보:
  - 이름: {user_name}
  - 현재 감정: {emotion}

  - 투자 초보자를 위한 쉬운 설명
  - 감정적 공감과 이해
  - 막연한 두려움 해소

  {emotion}한 상태의 사용자에게 적절한 톤으로 대화하세요."""

  def get_financial_advisor_prompt(user_level: str, interests: list) -> str:
      interests_str = ", ".join(interests) if interests else "일반 투자"

      return f"""당신은 {user_level} 수준의 투자자를 위한 금융 상담사입니다.

  관심 분야: {interests_str}

  조언 스타일:
  - {"기초적인 개념부터 차근차근" if user_level == "초보자" else "심화된 분석과 전략"}
  - 구체적인 투자 권유는 절대 금지
  - 위험성을 반드시 언급"""

  2단계: 점진적 YAML 분리 (나중에)

  # 기존 F-string 코드는 그대로 두고, 새로운 YAML 로더 추가
  class PromptManager:
      def __init__(self):
          self.use_yaml = False  # 개발 중에는 False

      def get_prompt(self, prompt_type: str, **kwargs) -> str:
          if self.use_yaml:
              return self._load_from_yaml(prompt_type, **kwargs)
          else:
              return self._load_from_fstring(prompt_type, **kwargs)

      def _load_from_fstring(self, prompt_type: str, **kwargs) -> str:
          # 기존 F-string 함수들 호출

  ✅ 빠른 개발

  - 코딩하면서 바로 테스트
  - IDE 자동완성과 디버깅 지원
  - 타입 힌트로 안정성 확보

  ✅ 점진적 마이그레이션

  - 작동하는 코드부터 시작
  - 필요할 때만 YAML로 분리
  - 기존 코드 영향 없음

  ✅ 학습 곡선 완만

  - Python 문법만 알면 시작 가능
  - YAML 문법 나중에 학습
  - 실제 필요성을 체감한 후 전환

  # my_app/ui/chatbot/langgraph_core/prompts/f_string_prompts.py

  def get_system_prompt(user_profile: dict = None) -> str:
      """기본 시스템 프롬프트 - F-string 버전"""
      name = user_profile.get("name", "") if user_profile else ""
      emotion = user_profile.get("emotion", "중립적") if user_profile else "중립적"

      return f"""당신은 FIREgen의 친근한 금융 AI 도우미입니다.

  사용자: {name}
  현재 감정 상태: {emotion}

  역할과 목표:
  - 투자 초보자도 이해할 수 있는 쉬운 설명
  - 사용자의 {emotion} 감정을 이해하고 공감
  - 투자에 대한 막연한 두려움 해소

  대화 원칙:
  - 따뜻하고 친근한 톤 유지
  - 전문용어 사용 시 반드시 쉬운 설명 추가
  - 투자 권유나 확실한 수익 보장 절대 금지"""

  def get_emotion_analysis_prompt(user_message: str) -> str:
