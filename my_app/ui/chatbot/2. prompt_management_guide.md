# AI 시스템 프롬프트 관리 가이드

## 개요

AI 개발자들이 실제 프로덕션 환경에서 사용하는 시스템 프롬프트 저장 및 관리 방법에 대한 종합 가이드입니다.

## 실제 AI 개발자들이 가장 많이 사용하는 방식

### 🏆 1위: YAML (Template-based)

**사용처:** Microsoft Prompty, LangChain, GitHub Models

**장점:**
- 📝 **최고의 가독성** - 긴 텍스트도 편하게 작성
- 💡 **구조화된 메타데이터** - 버전, 작성자, 설명 포함
- ⚙️ **템플릿 변수** - `{{variable}}` 형태로 동적 치환
- 🔧 **Git 친화적** - 버전 관리와 diff가 명확

**예시:**
```yaml
name: "Financial Advisor"
version: "1.2.0"
description: "AI 금융 상담사 시스템 프롬프트"
author: "FIREgen Team"
model:
  temperature: 0.7
  max_tokens: 1000
template: |
  당신은 {{domain}} 전문 금융 상담사입니다.
  
  사용자 정보:
  - 투자 경험: {{user_level}}
  - 현재 감정: {{emotion}}
  - 관심 분야: {{interests}}
  
  다음 원칙을 따르세요:
  - 초보자도 이해할 수 있게 설명
  - 감정적 공감 표현
  - 투자 권유 금지
inputs:
  - domain
  - user_level
  - emotion
  - interests
```

### 🥈 2위: Python Files (Dynamic)

**사용처:** 복잡한 프롬프트 로직, 조건부 생성

**장점:**
- 🔄 **동적 생성** - 런타임에 조건부 프롬프트 구성
- 🧪 **테스트 가능** - 단위 테스트 작성 용이
- 🔍 **IDE 지원** - 자동완성, 디버깅
- 📚 **주석 풍부** - 복잡한 로직 설명 가능

**예시:**
```python
def get_system_prompt(user_level: str, emotion: str, context: dict) -> str:
    """사용자 상태에 따른 동적 시스템 프롬프트 생성"""
    
    base = """당신은 친근한 금융 AI 도우미입니다."""
    
    # 경험 수준별 맞춤화
    if user_level == "beginner":
        base += "\n전문용어를 사용할 때는 반드시 쉬운 설명을 추가하세요."
    elif user_level == "expert":
        base += "\n상세한 기술적 분석과 심화 정보를 제공하세요."
    
    # 감정 상태별 톤 조절
    emotion_styles = {
        "불안": "따뜻하고 안정감을 주는 톤으로 대화하세요.",
        "호기심": "흥미롭고 교육적인 정보를 제공하세요.",
        "확신": "객관적이고 균형잡힌 관점을 제시하세요."
    }
    
    if emotion in emotion_styles:
        base += f"\n{emotion_styles[emotion]}"
    
    return base
```

## 주요 AI 회사들의 접근법

### Anthropic
- **5단계 계층구조**: Clear → Examples → Chain of Thought → XML Tags → Roles
- **XML 스타일 태그**: `<thinking>`, `<answer>` 등 구조화된 추론
- **30% 정확도 향상** 자동 프롬프트 엔지니어링 도구

### Microsoft
- **Prompty 프레임워크**: YAML 기반 표준화
- **멀티 프로바이더**: OpenAI, Azure OpenAI 등 지원
- **스키마 검증**: 입력값 유효성 검사

### LangChain
- **F-string 템플릿**: 보안성 강화 (Jinja2 대비)
- **파일 기반 로딩**: `from_file()` 메서드
- **채팅 프롬프트**: 다중 메시지 대화 지원

## FIREgen 프로젝트 적용 방안

### 현재 상황 분석
- LangGraph 기반 아키텍처
- Python 파일에 하드코딩된 프롬프트
- JSON 기반 PromptManager 존재

### 권장 마이그레이션 전략

#### 1단계: YAML 기반 템플릿 도입
```yaml
# prompts/system/base_personality.yml
name: "FIREgen 기본 성격"
version: "1.0.0"
description: "친근한 금융 AI 도우미의 기본 성격"
template: |
  당신은 친근한 금융 AI 도우미입니다.
  
  역할:
  - 투자 초보자를 위한 쉬운 설명
  - 감정적 공감과 이해
  - 막연한 두려움 해소
  
  대화 스타일:
  - 따뜻하고 친근한 톤
  - 전문용어는 쉽게 설명
  - 단계별 가이드 제공
```

#### 2단계: 동적 프롬프트 Python 함수
```python
# prompts/dynamic/context_prompts.py
def create_personalized_prompt(user_profile: dict, emotion_state: dict) -> str:
    """사용자 컨텍스트 기반 개인화 프롬프트 생성"""
    base_prompt = load_yaml_template("base_personality.yml")
    
    # 감정 상태 반영
    if emotion_state.get("emotion") == "불안":
        base_prompt += "\n특히 불안한 마음을 달래주는 따뜻한 톤으로 대화하세요."
    
    return base_prompt
```

#### 3단계: 통합 관리 시스템
```python
# prompt_manager_v2.py
class AdvancedPromptManager:
    def __init__(self):
        self.yaml_loader = YAMLPromptLoader()
        self.dynamic_generator = DynamicPromptGenerator()
    
    def get_prompt(self, prompt_type: str, context: dict = None) -> str:
        # YAML 템플릿 로드
        template = self.yaml_loader.load(prompt_type)
        
        # 동적 컨텍스트 적용
        if context:
            return self.dynamic_generator.apply_context(template, context)
        
        return template
```

## 베스트 프랙티스

### 1. 버전 관리
- 모든 프롬프트에 버전 정보 포함
- 의미있는 변경사항은 메이저 버전 업
- 호환성 유지를 위한 deprecated 마킹

### 2. 테스팅
- 프롬프트별 테스트 케이스 작성
- 예상 출력과 실제 출력 비교
- 성능 벤치마크 설정

### 3. 보안
- 사용자 입력 검증
- 프롬프트 인젝션 방지
- 민감 정보 하드코딩 금지

### 4. 성능 최적화
- 토큰 수 최적화
- 캐싱 전략 수립
- 응답 시간 모니터링
