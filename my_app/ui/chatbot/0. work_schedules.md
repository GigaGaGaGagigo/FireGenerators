#### **Day 13 (8/16, 금): LLM의 변덕에 대비하기**

- **🎯 오늘의 목표:** LLM이 항상 우리가 원하는 JSON 형식으로 응답하지 않을 수 있다는 사실에 대비하여, 프로그램이 다운되지 않도록 방어 코드를 추가한다.
    
- **💻 구현할 부분:**
    
    1. **JSON 파싱 예외 처리:**
        
        - langgraph_core/nodes/analysis_node.py의 analyze_input 함수 수정.
        - JsonOutputParser()를 호출하는 부분을 try-except 블록으로 감싼다.
        - except 블록에서는, 파싱에 실패했다는 로그를 남기고, analysis_result를 빈 딕셔너리 {}로 설정하여 반환. 이렇게 하면 뒷단의 노드들이 오류 없이 실행될 수 있다.
            
    2. **재시도 로직 (선택적 심화):**
        
        - langchain의 RetryWithErrorOutputParser를 사용하여, LLM이 잘못된 형식의 출력을 주면, "너의 출력은 JSON 형식이 아니야. 다시 해줘." 라는 피드백과 함께 자동으로 재시도를 요청하도록 구현.
