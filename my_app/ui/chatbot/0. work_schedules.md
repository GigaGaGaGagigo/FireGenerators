
### **Week 1: 코어 엔진 및 기본 UI 구축 (MVP)**

**목표: 터미널과 기본 UI에서 간단한 질문-답변이 가능한 뼈대 완성**

*   **Day 1: 프로젝트 환경 설정** ✅
    *   **작업:**
        1.  `my_app` 전체 디렉토리 구조 생성 (`mkdir`, `__init__.py` 파일 포함).
        2.  `requirements.txt` 파일 생성 및 `langgraph`, `google-generativeai`, `python-dotenv`, `pydantic` 설치.
        3.  **[개선안]** 프로젝트 루트에 `.env` 파일 생성 및 `GOOGLE_API_KEY` 등록.
        4.  **[개선안]** `config.py` 파일을 생성하여 `.env` 파일의 변수를 로드하는 설정 함수 구현.

*   **Day 2: LLM 클라이언트 및 상태 정의** ✅
    *   **작업:**
        1.  `langgraph_core/llm_client.py` 생성: `config.py`를 사용하여 Gemini 클라이언트를 초기화하는 싱글턴 또는 팩토리 함수 구현.
        2.  `langgraph_core/state/chat_state.py` 생성: `ChatState` 기본 구조 정의.
	        1. messages, user_profile 어떤 내용을 담아야 할지 몰라 기능을 구현하면서 추가할 예정

*   **Day 3: 최소 기능 그래프 구현**
    *   **작업:**
        1.  `nodes/response_node.py` 구현: `ChatState`를 받아 LLM을 호출하고 응답을 생성하는 가장 간단한 노드 작성.
        2.  `chat_graph.py` 구현: `response_node` 하나만 포함하는 가장 단순한 `StatefulGraph`를 정의하고 컴파일.
        3.  `ChatGraphManager` 클래스의 기본 골격 작성 (그래프 인스턴스화).

*   **Day 4: 터미널 기반 테스트**
    *   **작업:**
        1.  **[개선안]** 프로젝트 루트에 `main.py` (테스트 실행기) 생성.
        2.  `main.py`에서 `ChatGraphManager`를 호출하여 터미널에서 사용자 입력을 받고 응답을 출력하는 간단한 대화 루프 구현.
        3.  **성공 기준:** 터미널에서 "오늘 날씨 어때?"라고 물으면 LLM의 답변이 출력됨.

*   **Day 5: 기본 UI 연동**
    *   **작업:**
        1.  `my_app/ui/chatbot/chatbot.py` 파일 생성.
        2.  **[개선안]** `st.session_state`를 사용하여 `ChatGraphManager` 인스턴스를 관리하는 로직 추가.
        3.  `st.chat_input`으로 입력을 받고, `ChatGraphManager`를 호출하여 `st.chat_message`로 응답을 표시 (스트리밍 제외).
        4.  **성공 기준:** Streamlit 앱에서 질문을 입력하면 챗봇의 답변이 화면에 나타남.

### **Week 2: 지능형 노드 및 분석 기능 구현**

**목표: 사용자의 의도를 분석하고, 그 결과를 응답에 반영**

*   **Day 6: 분석/메모리 더미 노드 및 라우팅 구현**
    *   **작업:**
        1.  `nodes/conversation_node.py` 구현: "인사말인가?"와 같은 간단한 조건으로 분기하는 로직 작성.
        2.  `nodes/analysis_node.py`, `nodes/memory_node.py`를 빈 함수(상태를 그대로 통과)로 구현.
        3.  `chat_graph.py` 수정: `conversation_node`를 진입점으로 설정하고, 조건부 엣지(`add_conditional_edges`)를 추가하여 `analysis_node` 또는 `response_node`로 분기.

*   **Day 7: 분석 노드 구현 (감정 분석)**
    *   **작업:**
        1.  `prompts/analysis_prompts.py` 생성 및 감정 분석용 프롬프트 작성.
        2.  **[개선안]** 분석 결과(예: `{ "emotion": "불안", "confidence": 0.9 }`)를 파싱할 Pydantic 모델 정의.
        3.  `analysis_node.py` 수정: LLM을 호출하고, 결과를 Pydantic 모델로 파싱하여 `ChatState`에 저장. JSON 파싱 에러 처리 로직 추가.

*   **Day 8: 분석 노드 강화 (키워드 추출)**
    *   **작업:**
        1.  `analysis_prompts.py`의 프롬프트를 수정하여 감정과 핵심 키워드를 동시에 추출하도록 개선.
        2.  Pydantic 모델을 확장하여 키워드 필드 추가.
        3.  `analysis_node.py` 로직 업데이트.
        4.  **[개선안]** `tests/` 디렉토리 생성 및 `pytest` 설치. `analysis_node`가 Mock LLM 응답을 정확히 파싱하는지 검증하는 단위 테스트(`test_analysis_node.py`) 작성.

*   **Day 9: 개인화 응답 노드 구현**
    *   **작업:**
        1.  `prompts/system_prompts.py` 파일 생성 및 기본 시스템 프롬프트 정의.
        2.  `response_node.py` 수정: `ChatState`에 저장된 분석 결과(감정, 키워드)를 시스템 프롬프트에 포함하여 개인화된 응답을 생성하도록 개선.
        3.  **성공 기준:** "AI 주식이 무서워요" 입력 시, 분석 노드가 '불안', 'AI', '주식'을 감지하고, 응답 노드가 "불안한 마음에 공감합니다. AI 주식 분야는..."과 같이 맞춤형 답변을 생성.

*   **Day 10: UI 스트리밍 응답 적용**
    *   **작업:**
        1.  **[개선안]** `ChatGraphManager`가 `graph.astream_events`를 사용하도록 수정. `response_node`에서 생성되는 토큰을 실시간으로 `yield`.
        2.  **[개선안]** `chatbot.py`에서 `st.write_stream`을 사용하여 스트리밍 응답을 화면에 표시.
        3.  **성공 기준:** Streamlit UI에서 답변이 타이핑되듯이 실시간으로 나타남.

### **Week 3: 데이터 영속성 (메모리) 구현**

**목표: Supabase 연동을 통해 사용자의 정보를 기억하고 대화에 활용**

*   **Day 11: 데이터베이스 설계 및 설정**
    *   **작업:**
        1.  Supabase 프로젝트 생성.
        2.  **[개선안]** `user_profiles`, `conversation_history` 테이블 생성. `user_id`, JSONB 타입 필드, `created_at`, `updated_at` 등 상세 스키마 적용.
        3.  `.env` 파일에 Supabase URL 및 키 추가, `config.py`에 로드 로직 추가.

*   **Day 12: Supabase 매니저 구현**
    *   **작업:**
        1.  `utils/supabase_client.py` 생성.
        2.  **[개선안]** `SupabaseManager` 클래스 구현. 모든 DB I/O 메서드(`load_profile`, `save_profile` 등)를 `async def`로 작성.

*   **Day 13: 메모리 노드와 DB 연동**
    *   **작업:**
        1.  `prompts/memory_prompts.py` 생성 및 대화 요약 프롬프트 작성.
        2.  `memory_node.py`를 `async`로 수정. LLM으로 대화를 요약하고, 분석된 키워드와 함께 `SupabaseManager`를 호출하여 사용자 프로필을 DB에 저장(UPSERT).

*   **Day 14: 비동기 그래프 전체 적용**
    *   **작업:**
        1.  `ChatGraphManager`의 실행 메서드를 `ainvoke` 또는 `astream_events`로 완전히 전환.
        2.  `ChatGraphManager`가 `user_id`를 받아 세션 시작 시 `SupabaseManager`로 사용자 프로필을 로드하여 `ChatState`를 초기화하는 로직 추가.
        3.  **[개선안]** `tests/utils/test_supabase_client.py` 작성하여 실제 DB 연동 테스트 (통합 테스트).

*   **Day 15: UI 세션 관리**
    *   **작업:**
        1.  `chatbot.py`에 사용자 ID를 관리하는 로직 추가 (초기에는 `st.text_input`으로, 추후 인증 연동 가능).
        2.  페이지 새로고침 시에도 `st.session_state`의 `user_id`를 기반으로 대화 내용이 유지되도록 구현.
        3.  **성공 기준:** 브라우저를 새로고침해도 이전 대화 기록과 사용자 정보(예: "AI 주식에 관심 있으시군요")가 유지됨.
