### 1. 컨텍스트 엔지니어링을 활용한 F(감정, 대화)기반 사용자 데이터 생성 및 활용

**1.1. 서비스 정의**: 컨텍스트 엔지니어링을 활용하기 위한 기초 데이터를 생성합니다. 사용자의 관심사, 직업, 전문성, 취향 (관심 도메인)과 관련된 상품을 추천을 위한 데이터로 활용됩니다.
  
**1.2. 기존 기술과의 차이점**: 

  1. **개인화된 정보 기반 서비스**: 월 소득, 고정지출, 투자 유형 조사 등 수치화된 데이터로 추천하는 것이 아닌 개인화된 정보(감정분석, 관심사 등)로 유저 특화 상품을 추천하도록 컨텍스트 데이터 생성
  2. **멀티턴 챗봇**: LLM과의 자연스러운 대화를 활용해 개인화된 정보 수집 진행

**1.3. 사용자 워크 플로우:
-  페르소나: 26세 금융 초보자 

> 사용자 흐름:
> 1. 사용자는 앱을 설치하고 로그인한다.
> 2. 챗봇이 밝은 톤으로 "안녕하세요? 반갑습니다. OOO 님, 투자에 기대하는 바가 있으신가요?" 라고 인사한다.
> 3. 사용자는 "투자는 처음이에요. 경제 기사를 보면 어렵고 겁나요. 그래도 위험이 적은 AI 관련 주식에 투자 해보고 싶어요." 라고 답변한다. 
> 4. 시스템은 
> 	 - 감정분석: ‘조금 불안 + 호기심’ 
>      - 관심사 추출: ‘AI, 저위험’
>      - 투자 레벨: Beginner
>      - 투자 스킬: 0
>      - 감정 수치: -0.3
>     으로 분석한다.
>  5. 저장된 정보는 이후에 상품 추천이나, 관심 분야 뉴스 등 추천 콘텐츠에 활용된다.
>  6. 사용자는 원할 때마다 챗봇과 대화를 하여 저장된 컨텍스트 데이터를 업데이트한다.

**1.4. 시스템 워크 플로우**:

1.	사용자 ↔ LLM 대화
 - 시스템 프롬프트(역할·형식 지침)와 세션 state를 함께 LLM에 전달.
 - LLM이 답변을 생성하면 바로 화면에 표시.
2.	대화 전체를 Compact(요약)
 - 같은 LLM에게 “이번 턴 요약만 JSON으로 반환”을 요청.
 - 이 짧은 요약이 단기 Memory로 저장됨.
3.	spaCy 토큰화
 - 요약 텍스트를 spaCy 파이프라인에 전달해 형태소/엔터티 분리.
 - 한국어 기준 불용어‧조사를 제거해 핵심 토큰만 남김.
4.	중요 정보만 추려내기
 - 규칙: 고유명사, 감정 키워드, 투자·재무 관련 용어만 선택.
 - 점수화(빈도·위치)하여 상위 N개 키워드를 확정.
5.	DB 카테고리 저장(장기 Memory)
 - 관심사·직업·전문성·취향 컬럼에 키워드를 UPSERT.
 - 예) interest=“AI”, risk=“low”, emotion_score=-0.3 등.
6.	다음 턴 시작 시 컨텍스트 구성
 1) 단기 Memory: 직전 요약
 2) 장기 Memory: 프로필 키워드
 3) RAG 필요 여부 판단:
  -  사용자가 “최신”, “최근”, “오늘” 등을 언급 → 외부 문서 Top-3 검색 후 첨부.
  -  그 외엔 RAG 생략해 속도·비용 절감.
 4) Tools(예: 수익률 계산 API) 사양과 함께 다시 LLM에 전달.
 5) LLM이 답변과 업데이트할 필드를 JSON으로 반환하면, DB에 반영하고 1단계로 루프.
